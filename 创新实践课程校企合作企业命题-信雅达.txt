二、项目目标
（一）核心业务目标
（需明确项目解决的实际行业问题，量化功能与性能指标）
1.解决金融制度知识（银行监管政策、证券行业规范、保险制度等）查询效率低、精准度不足的问题，实现知识点快速检索与精准问答。
2.系统覆盖至少 1000+ 条金融制度核心知识点，支持文本、PDF、图片（OCR 解析）文档导入，问答响应时间≤2 秒，答案准确率≥85%（人工校验标准）。
3.支持关键词检索、自然语言问答、条文关联查询三种核心场景，满足金融从业者、高校学生的学习与工作需求。
（二）能力培养目标
（需对应课程要求，明确学生通过项目需掌握的能力）
1.系统分析与设计能力：掌握需求拆解、模块划分、架构设计的方法，能完成 RAG 系统从需求到设计的全流程规划。
2.技术应用与创新能力：熟练使用指定技术栈，实现至少 1 处功能或技术创新（如知识库增量更新、答案可信度评分）。
3.团队协作与沟通能力：通过分工协作完成项目，提升跨角色沟通效率，能清晰同步项目进展与问题。
4.文档撰写与答辩能力：规范完成各阶段技术文档，清晰展示项目成果，准确回应技术疑问。
3. 课程思政目标
（需融入科技报国、工匠精神、职业道德等导向）
1.树立知识产权保护意识，确保项目代码与方案原创性，不盗用第三方核心技术。
2.培养精益求精的研发态度，注重系统稳定性与用户体验优化，严谨处理金融制度数据的准确性。
3.增强科技强国使命感，对标金融科技领域前沿技术，分析国内 RAG 应用与国际先进水平的差距及改进方向。
三、技术要求
（一）核心技术栈
技术类别	推荐技术 / 工具	备注（版本、选型要求等）
编程语言	Python	3.8 及以上版本，适配主流 AI 框架
开发框架	LangChain、FastAPI	LangChain 搭建 RAG 架构，FastAPI 开发后端接口
数据库 / 存储	MySQL、Milvus	MySQL 存储结构化数据，Milvus 作为向量数据库
部署 / 测试工具	Docker、pytest、Postman	Docker 环境打包，pytest 单元测试，Postman 接口调试
其他关键技术	Tesseract（OCR）、开源 LLM（通义千问 / ChatGLM）	OCR 解析图片类文档，LLM 选择开源可商用版本
（二）技术具体要求
1.知识库构建：实现金融制度文档自动解析、分段、embedding 生成与向量存储，支持知识点标签分类管理。
2.检索与问答：结合向量检索与关键词检索，通过 LangChain 整合 LLM 生成通顺、准确的问答结果，支持答案溯源（标注对应制度条文）。
3.交互与部署：开发 Web 端前端交互界面，完成系统 Docker 容器化部署，确保跨环境可运行。
